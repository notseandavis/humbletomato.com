BMAD

Breakthrough Method for Agile AI-Driven Development. Specialized agent personas guide you through the full lifecycle with structured workflows grounded in agile practices.

Source: GitHub · Docs · Created by BMad Code (open source, no paywalls)

Also known as: Build More Architect Dreams

The Core Idea

Different phases of development need different expertise. Requirements analysis needs different thinking than architecture design. Architecture needs different thinking than implementation.

BMAD models this through specialized AI agent personas. Instead of one generalist agent doing everything, you work with experts:

  • Analyst for requirements
  • PM for roadmaps and prioritization
  • Architect for system design
  • Developer for implementation
  • UX Designer for interfaces
  • Scrum Master for process guidance
  • QA Engineer for testing strategies

12+ personas total. Each has specific prompts, workflows, and outputs tuned to their role.

How It Works

Install via npm:

npx bmad-method install

Invoke a persona:

bmad analyst        # Start requirements gathering
bmad architect      # Design system architecture
bmad developer      # Begin implementation

Or use Party Mode to get multiple personas collaborating in one session:

bmad party --roles analyst,architect,pm

The bmad-help skill provides guidance at any point. Stuck? Run bmad-help and the system suggests what to do next based on current project state.

Scale-Domain-Adaptive Planning

Small projects don't need enterprise-level documentation. Big projects do.

BMAD adjusts automatically:

Project Scale Planning Depth Artifacts Generated
Side project Light README, basic architecture doc
Startup MVP Medium PRD, architecture decision records, API specs
Enterprise product Comprehensive Full spec suite, compliance docs, deployment runbooks

You tell BMAD the scale during setup. It adjusts workflow complexity accordingly.

Module Ecosystem

BMAD is modular. Core installation gives you the base method. Additional modules add specialized capabilities:

Module Purpose
BMM (Core) Base methodology and workflows
BMB (Builder) Extended development workflows
TEA (Test Architect) Advanced testing strategies
BMGD (Game Dev Studio) Game-specific workflows and personas
CIS (Creative Intelligence) Design and creative workflows

Install what you need. Skip what you don't.

Philosophy

"AI agents as expert collaborators who guide YOU through structured process. Not doing thinking for you, bringing out your best thinking."

The personas ask questions. They surface tradeoffs. They suggest approaches. But you make decisions.

This is different from "agent does everything while you watch." BMAD assumes you're an active participant. The structure exists to make your thinking more rigorous, not to replace it.

Party Mode in Practice

Example session:

You: "We need to add user authentication"

Analyst: "What auth methods? Email/password, OAuth, both?"

Architect: "If OAuth, which providers? How do we handle 
           provider outages? Fallback to email/password?"

PM: "This affects signup conversion. Do we require email 
     verification? Adds friction but improves quality."

UX Designer: "Social login buttons increase conversion but 
              raise privacy concerns. How do we message that?"

You get multiple expert perspectives in one conversation. Make an informed decision, then the Developer persona implements based on that decision.

When to Use BMAD

You're building something complex that benefits from structured planning. You have (or will have) a team and want role-based workflows. You value comprehensive lifecycle coverage over minimalism.

You want proven agile practices baked into your AI workflow. You're comfortable with more files and ceremony in exchange for thoroughness.

When to Look Elsewhere

You're a solo builder who wants maximum simplicity. You value speed over comprehensive documentation. You don't want to learn a large system upfront. You prefer lightweight, opinionated workflows.

In those cases, consider GSD or similar methods that optimize for solo velocity over team coordination.

BMAD vs GSD

Both solve spec-driven development. Different philosophies:

Aspect GSD BMAD
Workflow 4 commands 34+ workflows
Personas One agent, task-focused 12+ role-based personas
Structure Minimal, hidden complexity Explicit, comprehensive
Best for Solo builders, simple projects Teams, complex projects
Learning curve Gentle Steeper
Modularity Opinionated, all-or-nothing Modular, pick components

Neither is universally better. Pick based on your project complexity, team size, and preference for simplicity versus comprehensiveness.
