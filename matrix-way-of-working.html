Matrix Way of Working

How the Context Matrix changes daily development practice. Daily releases, evolving specs, and new roles.

A Week With Matrix

# Day 1: Write a rough spec, get running code
$ cat specs/checkout.md    # L0 sketch: bullet points, intent only
$ matrix build             # Agent generates R0 from spec
$ npm test                 # 3 of 12 E2E fail → spec has gaps

# Day 2: Fix the spec, not the code
$ vim specs/checkout.md    # Add missing edge cases from test failures
$ matrix build             # Agent regenerates R1 from updated spec
$ npm test                 # 10 of 12 pass

# Day 3: Converge
$ vim specs/checkout.md    # Tighten error handling section
$ matrix build             # R2
$ npm test                 # 12 of 12 pass → RC0

# Day 4: Product reviews RC0, requests copy change
$ vim specs/checkout.md    # Update error message wording
$ matrix build && npm test # Green → ship it

Rejection is cheap. The agent rebuilds from updated specs in hours. Nobody spent three weeks on R1, so nobody fights to ship it despite known issues.

The PRD and code converge together. Neither is "done" first. Both are done when E2E passes.

E2E Tests as Definition of Done

// The agent's job isn't "build the feature"
// It's "make the tests green"
describe('checkout', () => {
  it('applies discount code', async () => {
    await page.fill('#discount', 'SAVE20');
    await page.click('#apply');
    expect(await page.textContent('#total')).toBe('$80.00');
  });

  it('shows error for expired code', async () => {
    await page.fill('#discount', 'EXPIRED');
    await page.click('#apply');
    expect(await page.textContent('#error')).toBe('This code has expired');
  });
});

The spec template defines expected behavior. E2E tests encode it as assertions. Tests pass or they don't. No ambiguity.

Spec Quality Levels

L0: SketchBullet points, intent onlyScaffolding + questions
L1: DraftStructured sections, some gapsCompilable code
L2: SpecifiedComplete, edge cases notedPassing unit tests
L3: VerifiedValidated by successful runsPassing E2E tests

Start at L0. Let R0 tell you what's missing. A rough spec that runs today teaches more than a perfect spec next month.

How Roles Change

PMs become builders. The person writing the spec shapes what gets built. Product judgment is the primary input to the build loop.

Engineers become architects. They own the templates, architecture docs, and infrastructure. They debug the loop, not the code.

What This Is Not

Not waterfall with faster typing. Speed is wasted if inputs don't iterate at the same pace.

Not "AI replaces product." Product judgment matters more, not less. Someone still decides what the error message says.

Not "AI replaces architecture." Templates and architecture docs are the machine. Engineers keep the machine running.
