The Solution Space

Vague descriptions produce multiple valid answers. Add constraints and the possibilities shrink. This is why specs matter.

When you describe something imprecisely, you create room for different valid interpretations — multiple correct answers.

Every constraint narrows the range. Remove constraints and the solution space expands. Add them and it shrinks.

Here's what this looks like in code.

Example 1: Sort a List

Spec: "Write a function that sorts a list"

Valid solution A:

def sort_list(items):
    return sorted(items)

Valid solution B:

def sort_list(items):
    # Bubble sort
    n = len(items)
    for i in range(n):
        for j in range(0, n-i-1):
            if items[j] > items[j+1]:
                items[j], items[j+1] = items[j+1], items[j]
    return items

Valid solution C:

def sort_list(items):
    # Quicksort
    if len(items) <= 1:
        return items
    pivot = items[len(items) // 2]
    left = [x for x in items if x < pivot]
    middle = [x for x in items if x == pivot]
    right = [x for x in items if x > pivot]
    return sort_list(left) + middle + sort_list(right)

All three work. All three are correct. The spec didn't say which one.

Add constraint: "Sort in-place, O(n log n)"

Now solution A is out (not in-place). Solution B is out (O(n²)). Solution C is out (creates new lists).

Valid solution:

def sort_list(items):
    # In-place heapsort
    def heapify(arr, n, i):
        largest = i
        left = 2 * i + 1
        right = 2 * i + 2
        
        if left < n and arr[left] > arr[largest]:
            largest = left
        if right < n and arr[right] > arr[largest]:
            largest = right
            
        if largest != i:
            arr[i], arr[largest] = arr[largest], arr[i]
            heapify(arr, n, largest)
    
    n = len(items)
    for i in range(n // 2 - 1, -1, -1):
        heapify(items, n, i)
    for i in range(n - 1, 0, -1):
        items[0], items[i] = items[i], items[0]
        heapify(items, i, 0)
    return items

The solution space shrunk from dozens to a handful. Heapsort, in-place mergesort, maybe a few others.

Add constraint: "Stable sort"

Now heapsort is out (not stable). You're left with very few options. In-place mergesort or a specialized variant.

Each constraint eliminated implementations. That's the pattern.

Example 2: Fetch Data

Spec: "Fetch user data from API"

Valid solution A:

async function getUser(id) {
    const response = await fetch(`/api/users/${id}`);
    return response.json();
}

Valid solution B:

async function getUser(id) {
    const response = await axios.get(`/api/users/${id}`);
    return response.data;
}

Valid solution C:

function getUser(id, callback) {
    const xhr = new XMLHttpRequest();
    xhr.open('GET', `/api/users/${id}`);
    xhr.onload = () => callback(JSON.parse(xhr.responseText));
    xhr.send();
}

Different libraries. Different patterns. All fetch data.

Add constraint: "Handle network errors gracefully"

async function getUser(id) {
    try {
        const response = await fetch(`/api/users/${id}`);
        if (!response.ok) {
            throw new Error(`HTTP ${response.status}`);
        }
        return response.json();
    } catch (error) {
        console.error('Failed to fetch user:', error);
        return null;
    }
}

Now you need error handling. The structure is more defined.

Add constraint: "Retry up to 3 times with exponential backoff"

async function getUser(id) {
    const maxRetries = 3;
    let attempt = 0;
    
    while (attempt < maxRetries) {
        try {
            const response = await fetch(`/api/users/${id}`);
            if (!response.ok) {
                throw new Error(`HTTP ${response.status}`);
            }
            return response.json();
        } catch (error) {
            attempt++;
            if (attempt >= maxRetries) {
                console.error('Max retries exceeded:', error);
                return null;
            }
            const delay = Math.pow(2, attempt) * 1000;
            await new Promise(resolve => setTimeout(resolve, delay));
        }
    }
}

The solution space collapsed. There's basically one way to do exponential backoff with three retries. You can bikeshed the details, but the structure is set.

Example 3: Store Preferences

Spec: "Store user theme preference"

Valid solution A:

function saveTheme(theme) {
    localStorage.setItem('theme', theme);
}

Valid solution B:

function saveTheme(theme) {
    document.cookie = `theme=${theme}; max-age=31536000`;
}

Valid solution C:

async function saveTheme(theme) {
    await fetch('/api/preferences', {
        method: 'POST',
        body: JSON.stringify({ theme })
    });
}

Valid solution D:

function saveTheme(theme) {
    fs.writeFileSync('user-prefs.json', 
        JSON.stringify({ theme }));
}

Client-side, server-side, file system. All store the preference.

Add constraint: "Persist across devices"

Solutions A, B, and D are out. Only server-side storage works.

async function saveTheme(theme) {
    await fetch('/api/preferences', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ theme })
    });
}

Add constraint: "Real-time sync without polling"

// Client
const ws = new WebSocket('wss://api.example.com/sync');

function saveTheme(theme) {
    ws.send(JSON.stringify({ 
        type: 'update-preference',
        key: 'theme',
        value: theme 
    }));
}

ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    if (data.type === 'preference-updated') {
        applyTheme(data.value);
    }
};

Now you need WebSockets or Server-Sent Events. REST won't cut it. The constraint eliminated entire categories of solutions.

Why This Matters

When you tell an AI to build something, every detail you omit expands the solution space. The AI picks one valid answer, but you might have expected a different (equally valid) one.

This isn't the AI being stupid. It's math. Fewer constraints means more valid solutions.

In linear algebra, a system with fewer equations than unknowns has infinite solutions. In code, a spec with fewer constraints than decisions has multiple valid implementations.

Want predictable outputs? Shrink the solution space. Add constraints until there's basically one answer left.

That's why spec-driven development works. Not because it makes AI smarter, but because it eliminates degrees of freedom.

The Pattern

Vague spec → wide solution space → surprising outputs

Precise spec → narrow solution space → predictable outputs

The solution space isn't a bug to fix. It's the natural consequence of incomplete information. Be intentional about which parts you nail down and which you leave open.
