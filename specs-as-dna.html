Specs as DNA

Specs encode intent like DNA encodes organisms. Agents express that intent as code. Different environments produce different implementations from the same spec.

# Spec (the "DNA")
# User Story: Display User Profile
- Show user's name, email, join date
- Format join date as "Member since MMM YYYY"
- If user has no email, show "No email on file"

Same spec, two frameworks. The spec is framework-agnostic. Architecture context determines output.

// React expression
function UserProfile({ user }) {
  const joinDate = new Date(user.joinedAt)
    .toLocaleDateString('en-US', { month: 'short', year: 'numeric' });
  return (
    <div className="profile">
      <h2>{user.name}</h2>
      <p>{user.email || "No email on file"}</p>
      <p>Member since {joinDate}</p>
    </div>
  );
}

Vague Spec vs. Constrained Spec

# Bad: vague DNA
# Feature: User Settings
Users need to be able to change their preferences.
Make it easy to use.
# What preferences? What UI? What storage? Unpredictable output.
# Good: constrained DNA
# Feature: User Settings

## Data Model
- user_preferences table
  - theme: 'light' | 'dark' | 'auto'
  - email_notifications: boolean
  - language: ISO 639-1 string

## API Contract
POST /api/user/preferences
Body: { theme, email_notifications, language }
Returns: 200 + updated object | 400 + validation errors

## Acceptance Criteria
- Theme change updates UI without reload
- Invalid language shows inline error
- Unsaved changes warning on tab close

Define what and constraints, not implementation details.

Clean specs produce predictable output. Bad specs produce malformed output. Same as DNA and proteins.

Don't Pre-Synthesize Your Specs

Tempting: merge your PRD + architecture docs into one clean file, then feed it to the agent. Don't.

The Data Processing Inequality says processing can only destroy information, never create it: I(X; Z) ≤ I(X; Y)

What you lose by synthesizing:

  • Redundancy that gives the agent multiple angles on a concept
  • Structural context from how the original doc is organized
  • Edge cases that get trimmed during summarization

Exception: synthesis that adds new information (resolving ambiguities, deriving constraints from combining two docs, resolving contradictions). If you're just reformatting, feed the raw files.
