AI Coding Tools: Quick Reference

Which tool for which workflow in 2026. Terminal agents, IDE integrations, and structured frameworks compared.

The AI coding ecosystem split into three categories: terminal agents (autonomous debugging, infra work), IDE integrations (visual feature building), and structured frameworks (spec-driven workflows). Pick by workflow, not hype.

Decision Guide

# Scenario: "I have a legacy bug in my auth middleware"
→ Claude Code (autonomous trace + fix)

# Scenario: "Build a new dashboard page with 3 charts"
→ Cursor (visual interface, Composer mode)

# Scenario: "Migrate entire DB schema from Prisma to Drizzle"
→ Aider (surgical precision, git-aware commits)

# Scenario: "Build a full product from PRD, solo developer"
→ GSD (fresh context per task, wave parallelism)

# Scenario: "Complex greenfield, team needs coordination"
→ BMAD (full lifecycle, document handoffs, audit trail)

# Scenario: "Move from vibe coding to structured specs"
→ GitHub Spec Kit (spec → plan → tasks → implement)

Key Differences

Claude Code: Autonomous execution (~20 consecutive tool calls). Reads files, plans changes, executes, runs tests, commits. Best SWE-bench score (77.2%). Terminal-native. Extended Thinking mode for complex reasoning. $20-200/mo.

Cursor: GUI powerhouse. Composer mode for multi-file editing. "Mind-reading" autocomplete. Codebase indexing (RAG-like). Privacy Mode + SOC 2 certified. Context compression under load (200K → 70K-120K). 7-15GB RAM consumption. $20-200/mo.

Aider: Git-aware editing (automatic commit per change). 80-98% token reduction vs naive approaches. Voice-to-code support. Three modes: Ask, Code, Architect. Works with local models (Ollama). $0.01-0.10 per feature with GPT-4o.

GSD: Meta-prompting system. Fresh 200K token window per task. Wave-based parallelism. Four commands: new-project, discuss-phase, build, verify. Token-hungry but prevents degradation. Covered here.

BMAD: Agent-as-code framework. Seven specialized personas (Analyst, PM, Architect, Developer, SM, QA, Orchestrator). Sequential phases with document handoffs. Party Mode for multi-persona sessions. Covered here.

GitHub Spec Kit: Open-source toolkit (28K+ stars). CLI + templates + prompts. Works with 22+ AI agents. Workflow: spec → plan → tasks → implement → review → merge. Addresses "vibe coding" drift.

Hybrid Workflows

2026 pattern: Cursor for UI + Claude Code in integrated terminal for backend/infra work. Or Aider for surgical edits + GSD for greenfield features. Tools are complementary, not exclusive.

Secret sauce: CLAUDE.md or .cursorrules files store project preferences ("Use Functional Components only", "Always use Zod for validation"). Agents read these as context.

Tool-Agnostic Principles

These patterns work regardless of tool:

  • Context Matrix — Compose specs + architecture → implementation
  • Ralph Loop — Fresh context per iteration prevents degradation
  • Spec-Driven Development — Specs as source of truth, code as compiled output
  • Solution Space — Constraints eliminate valid alternatives
  • Telephone Game — Fewer LLM steps = less information loss

Good tools implement these principles. Great developers understand the principles behind the tools.
